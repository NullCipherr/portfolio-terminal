# Contributing

Thank you for your interest in contributing to Portfolio Terminal.

## Development Setup

1. Fork the repository.
2. Create a feature branch from `main`.
3. Install dependencies with `npm install`.
4. Run the project with `npm run dev`.

## Pull Request Guidelines

- Keep PR scope focused and easy to review.
- Include a clear description of behavior changes.
- Update relevant documentation when architecture or behavior changes.
- Ensure `npm run lint` and `npm run build` pass before opening PR.

## Commit Recommendations

Use clear and action-oriented commit messages, for example:

- `feat: add project search command`
- `fix: prevent command execution while content is loading`
- `docs: update ssot integration section in readme`

## Reporting Issues

Please use issue templates and include:

- expected behavior;
- actual behavior;
- reproduction steps;
- environment details.

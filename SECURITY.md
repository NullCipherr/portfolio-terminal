# Security Policy

## Supported Versions

Only the latest `main` branch is actively supported for security updates.

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

Instead, contact the maintainer privately with:

- vulnerability description;
- impact assessment;
- reproduction steps;
- suggested remediation (if available).

You can use GitHub Security Advisories (preferred) or direct private contact.

## Response Timeline

- Initial acknowledgment: up to 72 hours.
- Triage and severity assessment: up to 7 business days.
- Fix timeline: depends on impact and complexity.
